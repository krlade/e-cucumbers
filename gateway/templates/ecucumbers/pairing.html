Jak to działa?

1 Użytkownik w API generuje token parowania (POST /api/nodes/pairing-token/).
2 Gateway wysyła token + swoje device_id do API i otrzymuje JWT.
3 Co {{ heartbeat_interval }}s gateway odpytuje API (POST /api/nodes/heartbeat/) i pobiera zakolejkowane komendy.
4 Komendy (np. TURN_ON, WATER_PUMP_ON) są tłumaczone na komunikaty MQTT i wysyłane do urządzeń peryferyjnych.
